Test task for Back-end Developer fans-crm.com
Before completing the task, you should understand that it is not necessary to complete the entire task in full. It is enough to build the basis of the application and describe all components and methods.
In addition, you should be ready to explain during a technical interview how it works.
Task:
The code must be available on GitHub for review. Build a simple server application according to the requirements:
Framework: NestJS
Node.js version: v22+
Database: MongoDB
ORM: Mongoose

The application should be a simple REST API:
Sending data via POST request, saving to database and output to console as logs
Receiving data via GET request

Use the following addresses:
/api/v1/add-user
/api/v1/get-users/ (must have pagination and optional filter by username, email or phone)
/api/v1/get-user/:id

You need to generate random user data (name, email, phone). You should also organize authorization of requests using JWT token in HTTP header.
At the start, if the database is empty, the backend should create 2 million random user records (name, email, phone, date of birth), and queries should work with this data.
Bonus task (for those who apply for full-stack):
Create a simple desktop application with the following requirements:
Framework: Electron
Node.js version: v22+
TypeScript

The application should be a simple frontend client on ReactJS, with the ability to switch between two pages via links. One of these pages should receive and send data to your backend. For example, save and retrieve user data.
